.safetensors file format

a performant file format for ML and AI models

by Charles Iliya Krempeaux

The safetensors file format is a performant file format for machine learning (ML) and artificial intelligence (AI) models.

A big part of the performance of the format is because its contents are meant to be memory mapped (mmap) (rather than parsed or copied).

⚠️ But, there is big caveat with the safetensors file format. It is only a performant on computers with little-endian hardware. This is because all the numbers in th safetensors file format are stored in little-endian order.

File Layout

The safetensors file format does not have magic bytes that let you identify it as a safetensors file.

The safetensors file format has 2 sections:

  1. header
  2. body

      3. The header is a 64-bit p-string.

      I.e., the first 8 bytes (i.e, the first 64-bits) of the safetensors file format header is a little-endian 64-bit unsigned integer. (On a machine with little-endian hardware in the Go programming language (golang) this is the uint64 type — but only on a computer with little-endian hardware.)

      The value of this little-endian 64-bit unsigned integer is the length of the header bytes (measure in bytes).

      The header bytes is JSON.
